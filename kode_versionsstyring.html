Versionsstyring

Hvad er det?

Versionsstyring er en metodisk tilgang. Det kan være noget så simpelt som navngivning af ens filer eller mapper og suppleres med notater om ændringer der sker fra version til version.

Version Control System (VCS) understøtter versionsstyring. Denne software holder styr på de forskellige versioners forskelle. Det kan være en tjeneste i skyen eller et program installeret på en lokal server.

De mest udbredte måder er Git-protokollen og SVN (Subversion).

Hvorfor er det relevant?

  • Man kan undgå fejl.
  • Man kan gå tilbage, hvis fejlene opstår.
  • Man kan holde styr på delprojekter - samme ting kan laves i forskellige versioner til forskellige kunder.
  • Man kan samarbejde.

Hvor bruges det?

Det bruges i alle typer af IT-projekter og af alle som arbejder professionelt med IT-udvikling – lige fra hobby-kodere til store organisationer.

Det kan dog også bruges til at dele andet end kun kildekode – fx aftaler, rapporter osv.

Git

  • En standard - en protokol
  • Versionsstyringssystem
  • Man kan selv opsætte en Git-server eller bruge en online tjeneste - fx GitHub, GitLab, BitBucket.

GitHub

  • Webplatform
  • Oprettelse af Git-repositories i skyen
  • Man behøver ikke installere en server
  • Har flere features indbygget – fx til samarbejde, projektstyring osv.
  • Kodedelings-site – et slags socialt medie for kodning
  • Portfolio for kode
  • Kommandolinje eller desktopprogram
  • Man kan hente kode til egne projekter
  • Mulighed for private repositories

Væsentlige begreber

  • Repository (repo) – samling af filer, projektmappe
  • Branch – forgrening, parallel version af repo som ikke har betydning for den oprindelige
  • Master branch – hovedforløbet, den oprindelige version, den der gælder
  • Commit – revision, milepæl; det svarer til at gemme lige netop det sted i versionen, som giver mulighed for at komme tilbage til det sted igen.
  • Pull Request – forslag til ny udgave, rettelser
  • Diff – forskel mellem to commits, difference
  • Merge – ændringer i en branch samles med en anden branch, fx gennem et pull request
  • Pull – man henter ændringer og merger dem
